# Аутентификация

## Цель
Описать текущие точки интеграции авторизации и где обновлять логику доступа.

## Ключевые моменты
- Проверки доступа выполняются в server components, server actions или API маршрутах.
- Конфигурация провайдеров и секреты задаются через переменные окружения.

## Что проверять при изменениях
- Ограничение доступа к приватным данным.
- Срок жизни сессий и обновление токенов.
- Поведение при ошибках авторизации.

## Связанные модули
- `src/app/api/` — маршруты, где проверяется доступ.
- `src/app/providers.tsx` или аналогичные провайдеры авторизации (если используются).

## Исторические материалы
Если в `docs/reports/` описана старая схема авторизации, считайте её исторической и сверяйте с текущей реализацией в коде.
