# API эндпоинты

## Цель
Зафиксировать, где находятся API маршруты и как поддерживать их в актуальном состоянии.

## Ключевые правила
- Новые маршруты добавляются в `src/app/api/`.
- Документируйте входные/выходные данные рядом с кодом (JSDoc/TS types).
- Учитывайте кэширование и ограничения по времени ответа.

## Что описывать
- Название маршрута и назначение.
- Требуемые параметры/тело запроса.
- Формат ответа и коды ошибок.

## Связанные модули
- `src/app/api/` — реализация маршрутов.
- `src/app/actions/` — server actions, если используются как альтернатива API.
- `prisma/` — доступ к данным.

## Исторические материалы
Старые сведения об API могут встречаться в `docs/reports/`; используйте их как справку, но сверяйтесь с кодом.
