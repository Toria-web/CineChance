# Настройка базы данных

## Цель
Зафиксировать шаги подключения PostgreSQL и обновления схемы при деплое.

## Ключевые шаги
1. Создать экземпляр PostgreSQL (например, Supabase/Neon/Render).
2. Настроить переменную `DATABASE_URL` в окружении.
3. Выполнить миграции: `npx prisma migrate deploy`.
4. Проверить доступность приложения и корректность запросов.

## Рекомендации
- Используйте пул подключений для продакшена.
- Следите за квотами и ростом данных.

## Связанные модули
- `prisma/` — схема и миграции.
- `docs/database/migrations.md` — процесс обновления схемы.

## Исторические материалы
Информация о ранних миграциях в `docs/reports/DB_MIGRATION_REPORT.md` является исторической.
