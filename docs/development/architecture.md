# Архитектура CineChance

## Цель
Описать верхнеуровневую архитектуру сервиса и основные технические компоненты, чтобы быстро ориентироваться в кодовой базе.

## Ключевые компоненты
- **Next.js приложение**: UI и серверные компоненты расположены в `src/app/`.
- **API маршруты**: серверные endpoints находятся в `src/app/api/`.
- **Слой данных**: Prisma схема и клиент лежат в `prisma/`, подключение настраивается через переменные окружения.
- **Статические ресурсы**: `public/` для ассетов, `src/app/globals.css` и Tailwind-конфигурация для стилей.

## Потоки данных
1. Клиентский UI вызывает server actions или API маршруты.
2. Серверный слой читает/пишет данные через Prisma.
3. Сервис рекомендаций использует данные профиля и историю просмотров (см. `src/app/recommendations`).

## Связанные модули
- `src/app/api` — API эндпоинты.
- `src/app/recommendations` — сценарии рекомендаций и UI.
- `prisma/` — схема БД и миграции.

## Исторические материалы
В `docs/reports/` хранятся отчёты и оптимизации прошлых итераций — детали могут отличаться от текущей реализации.
