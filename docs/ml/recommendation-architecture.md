# Архитектура рекомендаций

## Цель
Зафиксировать текущий подход к построению рекомендаций и места интеграции в продукт.

## Ключевые понятия
- **Источник данных**: история просмотров, лайки/дизлайки, профиль пользователя.
- **Ранжирование**: комбинирование жанров, рейтингов и исторических предпочтений.
- **Обогащение**: кэширование и фильтрация результатов перед выдачей.

## Поток работы
1. Сбор сигнала пользователя в UI.
2. Запрос к серверному слою (API или server actions).
3. Формирование рекомендаций и отдача на клиент.

## Связанные модули
- `src/app/recommendations/` — UI и серверная логика рекомендаций.
- `src/app/api/` — endpoints, если рекомендации вынесены в API.
- `prisma/` — хранение данных для обучения/ранжирования.

## Исторические материалы
Отчёты в `docs/reports/` содержат старые эксперименты и могут быть частично устаревшими.
