# Миграции базы данных

## Цель
Описать процесс безопасного изменения схемы данных и синхронизации с окружениями.

## Ключевые шаги
1. Обновить `prisma/schema.prisma`.
2. Создать миграцию: `npx prisma migrate dev --name <migration_name>`.
3. Проверить миграцию локально и в CI.
4. Для продакшена использовать `npx prisma migrate deploy`.

## Рекомендации
- Документируйте изменения в PR и указывайте затронутые модели.
- Для тяжёлых изменений готовьте план отката.

## Связанные модули
- `prisma/` — схема и миграции.
- `src/app/api/` — точки использования данных.

## Исторические материалы
`docs/reports/DB_MIGRATION_REPORT.md` содержит исторический отчёт о прошлых миграциях.
