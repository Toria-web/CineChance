# Схема данных

## Цель
Дать обзор ключевых сущностей и правил хранения данных.

## Ключевые понятия
- **Prisma schema**: источник правды для моделей и связей.
- **Миграции**: версионирование изменений структуры БД.
- **Индексы**: ускоряют выборки для рекомендаций и профиля пользователя.

## Где смотреть
- `prisma/schema.prisma` — определение моделей.
- `prisma/migrations/` — история миграций.

## Связанные модули
- `src/app/api/` — API доступ к данным.
- `src/app/actions/` — server actions, читающие/пишущие в БД.

## Исторические материалы
Отчёты о миграциях/оптимизациях в `docs/reports/` являются историческими; актуальная схема — в `prisma/`.
