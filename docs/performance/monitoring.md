# Мониторинг производительности

## Цель
Описать подход к отслеживанию производительности приложения и базы данных.

## Ключевые метрики
- Время ответа API и server actions.
- Время выполнения SQL запросов (slow queries).
- Ошибки и деградации на клиенте (например, длительный LCP).

## Практика мониторинга
- Локально: анализируйте логи запросов и профилируйте страницы, которые загружают рекомендации.
- В проде: используйте внешние инструменты мониторинга (Sentry/LogRocket/Datadog — в зависимости от текущих настроек).

## Связанные модули
- `src/app/api/` — места, где можно измерять время ответов.
- `prisma/` — оптимизация запросов и индексы.
- `docs/performance/SQL_OPTIMIZATION.md` — исторический кейс оптимизации.

## Исторические материалы
`docs/reports/` и `docs/performance/SQL_OPTIMIZATION.md` могут описывать прошлые проблемы, не все рекомендации актуальны.
